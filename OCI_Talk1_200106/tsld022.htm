Why should I use Oracle::OCI?

  • It brings together the best tools …

    • The power of the OCI API
    • The power of Perl language and extensive module library

  • Plus …

    • The DBI takes much of the grunt work out of OCI development
    • Oracle::OCI only needed for the more specialized code
    • DBD::Oracle, via the DBI, takes care of the rest

  • All of which leads to rapid application development

  • And… it’s fun!
